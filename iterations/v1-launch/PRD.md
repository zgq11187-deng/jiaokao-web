# 教考智联工作台 · PRD · v1-launch

> 文件位置：`iterations/v1-launch/PRD.md`
> 配合项目根目录的 `BRIEF.md` 阅读。这份 PRD 描述第一版上线要做的事。

## 1. 项目背景
教考智联工作台是面向专升本学生、任课老师和自学备考者的《计算机应用基础》教与学网站。第一版重点服务课堂教学配套和学生课后自学，让学生能按章节学习、刷题、模考，也让老师能备课、生成教学页、导出课堂材料。

参见：`../../BRIEF.md`

## 2. 本次目标
第一版上线后，学生能够登录并在授权后按章节完成学习、查看知识点、下载资料、刷题练习、查看错题和参加模拟考试。老师能够围绕章节完成备课、题库管理、布置练习、查看学习反馈，并通过 Qwen + Codex Agent + Notion 链路生成章节教学页。

第一版先服务你本人和一个班级学生，不做综合教育平台、不做学校教务系统、不做学生社交社区。

## 3. 用户与关键场景

### 场景 1：学生课前预习
学生登录后进入章节页面，查看本章学习目标、重点知识点和配套资料，带着问题进入课堂学习。

### 场景 2：学生课后复习
学生按章节复习知识点，下载讲义或课件，完成章节练习，并查看答案解析和错题。

### 场景 3：学生考前冲刺
学生进入模拟考试或综合练习，完成一套题后查看得分、解析和薄弱章节提示。

### 场景 4：老师备课
老师进入章节详情页，查看教案、课件、知识点、真题和易错点，用于准备课堂内容。

### 场景 4.1：老师同步 Notion 章节库
老师点击章节侧边栏刷新时，系统从 Notion 章节库同步所有章节到本地 SQLite。这样无论章节是在网页中新建，还是老师手动在 Notion 章节库中新建，都能出现在网站侧边栏，并参与后续原始页、A/B/C、导出等操作。

### 场景 5：老师布置练习
老师在教师工作台中选择章节题目或练习包，布置给学生作为课后任务。

### 场景 6：老师查看反馈
老师查看学生练习记录、错题集中点和章节掌握情况，用来调整下一次课的讲解重点。

### 场景 7：老师生成教学页
老师在章节详情页上传或选择图片、PDF、CSV、TXT、MD 等原始资料，系统用 Qwen 转成 Markdown 原始页，写入 Notion 原始页面库并关联章节；随后执行 Codex Agent A/B/C，生成考点、真题入库和章节教学页。教学页先写回 Notion 章节库，再同步到网页展示和导出。

Codex Agent C 生成的教学页必须对齐现有 Notion Agent 教学页样例格式：以“本节学习目标（识记 / 领会 / 应用）”开头，紧接讲稿口播，随后按 `---` 分页组织课堂导入、知识结构、重点难点、小节讲解、六段式概念深挖、历年真题演练、2026 新增题库、随堂练习、课堂小结、课后作业、下节预告和教师备课卡。真题答案解析必须保留可折叠的“答案与解析”入口，生成内容仍为草稿，需老师人工检查后才能发布或导出。

C 生成后必须进行结构校验：至少检查是否包含 `---` 分页、本节学习目标、知识结构、重点难点、真题折叠解析、课后作业和教师备课卡；校验问题写入生成日志和教学页 warning，不阻止保存草稿，但必须提示老师人工检查。

### 场景 8：老师先创建 Notion 讲义原始页
老师在章节详情页也可以不上传文件，直接为当前章节创建一个 Notion 讲义占位页。系统在 Notion 原始页面库创建标题为当前章节的原始课件页面，正文写入“未提取到文本内容，请检查 PDF 是否为扫描件”，并在 Notion 原始资料库创建关联记录；同时同步本地原始页面占位记录，作为后续 AIGC 教学页生成前的建页辅助流程。该流程不等同于 Qwen OCR，不调用 Qwen，也不绕过后续人工检查。

### 场景 9：老师扫描 Notion 触发项
老师可以在教师工作台手动扫描 Notion 复选框触发项。系统识别章节库的“自动填充考点”“生成课件”和原始资料库的“入库”复选框，按 Logseq 版专升本生成助手 Agent 规则执行 A/B/C：A 填充考点，B 真题入库，C 生成教学页；执行后清除触发复选框，并在 Notion 页面评论中汇报结果。评论或清勾失败只作为 warning 记录，AI 生成内容仍然必须人工检查后才能发布或导出。

## 4. 页面结构

- `/` 首页：说明网站定位，提供学生入口和教师入口。
- `/login` 登录页：学生、老师登录入口。
- `/register` 注册 / 申请访问页：学生提交注册申请，等待授权。
- `/chapters` 课程章节页：展示所有章节和学习进度入口。
- `/chapters/[id]` 章节详情页：展示章节学习内容、练习、资料和教学页生成流程。
- `/practice` 题库 / 练习页：按章节刷题、查看解析、回看错题。
- `/mock-exam` 模拟考试页：完成综合模拟题并查看解析。
- `/teacher` 教师工作台：管理章节、题库、练习、学生授权、学习反馈和教学页生成。
- `/resources` 资料下载页：集中下载课件、讲义、练习和复习资料。
- `/about` 关于课程页：说明课程适合谁、如何学习、老师如何使用。

## 5. 核心页面信息架构

### 首页 `/`
- 顶部介绍：网站名称、课程定位、学生入口、教师入口。
- 学生学习区：按章节学习、刷题练习、模拟考试、资料下载。
- 教师备课区：教案课件、题库管理、布置练习、学习反馈。
- 学习路径区：预习 -> 上课 -> 复习 -> 练习 -> 模考。
- 快速入口区：进入章节、进入练习、进入教师工作台。

### 登录 / 注册
- 登录页：账号、密码、登录状态提示。
- 注册页：学生提交姓名、手机号和必要说明。
- 授权状态：未授权用户不能访问课程内容、题库、模拟考试和教学页。
- 账号来源：支持学生自行注册等待审核，也支持老师提前录入账号。
- 审核方式：老师根据姓名 + 手机号判断，通过或拒绝访问。
- 学生账号管理：老师可查看所有学生账号，给学生开放权限、收回权限或删除学生账号；收回权限不删除学习记录，删除账号会移除该学生账号及其学习 / 答题记录，必须二次确认。

### 章节详情页 `/chapters/[id]`
- 本章学习目标。
- 核心知识点。
- 课件 / 讲义下载。
- 教师教案。
- 章节练习入口。
- 高频错题 / 易错点。
- 学生端章节详情首版直接读取本地章节、考点、真题和最新教学页数据；授权学生可以在同一页完成章节学习、章节练习、错题回看和导出资料下载。
- 学生端只显示老师开放给学生的章节；课程章节列表默认仅显示当前章节，点击“切换章节”后展开其他已开放章节，选择后自动收起，以减少移动端首屏占用。
- 学生端移动端页面顶部必须提供明显的当前章节与“切换章节”入口，确保学生在章节练习、错题回看、模拟考试和资料页也能快速切换已开放章节。
- 章节练习首版使用 `exam_questions` 中当前章节未隐藏题目，学生端应展示当前章节全部可练题，不额外截断为固定 20 题；学生提交后写入本地答题记录，系统按答案归一化判断正误，提交后展示正确答案、解析和知识点标签。模拟考试可以继续按配置数量抽题。
- 错题回看首版不单独维护错题库，而是从学生个人答题记录中聚合错误记录；学生可查看题干、自己的答案、正确答案、解析和最近错误时间。
- 模拟考试首版从本地题库按题型顺序抽取一组题，学生提交后写入答题记录，并展示得分、正确率、错题清单和薄弱章节提示；不代表真实考试成绩。
- 教学页生成区：
  - 老师端可手动同步 Notion 章节库中的章节到本地列表；该操作只同步章节标题、章、节、状态和 Notion page id，不读取章节正文、不导入习题，避免全量同步超时。学生端刷新只读取服务器中已开放章节，不直接触发 Notion 同步。
  - 老师可将章节“开放给学生”或“对学生隐藏”；新同步章节默认隐藏，Notion 中已删除的章节再次同步后先对学生隐藏，不直接物理删除本地题库、教学页和答题记录。
  - 上传或选择图片 / PDF / CSV / TXT / MD 原始资料。
  - Qwen 将资料转换为 Markdown 原始页。
  - Markdown 原始页写入 Notion 原始页面库，并关联当前章节；默认页面标题必须使用当前章节标题，不自动追加“原始页面”后缀；若当前章节下已存在同名原始页面，系统必须追加写入该同名页面，不再新建重复页面。历史上已经生成的“章节标题 + 原始页面”后缀页仅作为兼容迁移来源，不作为新的默认写入目标。
  - Markdown 原始页支持 1-6 级 Markdown 标题；写入 Notion 时因 Notion API 只支持 1-3 级标题，4-6 级标题统一按三级标题展示，避免 `####`、`#####` 原样显示成正文。
  - 支持无文件创建 Notion 讲义原始页：仅创建原始页面库占位页、原始资料库关联记录和本地占位记录，不调用 Qwen OCR。
  - Codex Agent A 自动填充考点。
  - Codex Agent B 筛选章节相关真题并入库；从 Notion 历年真题库读取候选题最多 150 条，按章节关键词预筛选后最多给 Codex 100 条，最终入库数量不另设上限。
  - Codex Agent C 生成章节教学页，版式对齐 Notion Agent 样例：学习目标、讲稿口播、课堂导入、知识结构、重点难点、按原小节展开、历年真题、2026 新增题库、随堂练习、课堂小结、课后作业、下节预告、教师备课卡。
  - C 生成教学页中的 Markdown 管道表格必须渲染为表格，知识结构图必须使用可显示的 Mermaid 图或降级为结构化列表；`<columns>` / `<column>` 双栏标记不能原样暴露给学生或老师。
  - 支持从教学页导入 Notion AI 生成的题库型习题：系统优先解析“章节习题 / 单章题库 / 本章练习 / 历年真题演练 / 2026 新增题库 / 模拟题 / 模拟考试 / 模拟训练 / 模拟演练 / 章节测试 / 习题精选 / 综合模拟题”等区块中的题目、选项、答案和解析，写入本地章节题库；“课堂提问 / 课堂互动 / 导入提问 / 课前提问 / 思考题”等课堂讨论题不默认导入学生练习。模拟题来源必须标记为“Notion AI 模拟题”，自编题来源标记为“Notion AI 自编题”，不得伪装成历年真题。导入时必须兼容“简答 / 操作题”混合分区、Notion callout / toggle 形式的题干与参考答案，并能在重复导入时纠正自编题误分类，例如带 A/B/C/D 选项且答案为多个字母的题应归为多选题，而不是操作题。
  - 老师选择当前章节后，可以单独点击“同步当前章节教学页”，系统只读取当前章节 Notion 页面正文并缓存到本地 `teaching_pages`，以前由 Notion Agent 直接写在章节页正文中的教学内容，也应在同步后显示到网页“章节教学页预览”中。
  - 老师同步当前章节教学页后，可以单独点击“导入当前章节习题”，系统解析“章节习题 / 单章题库 / 本章练习 / 历年真题 / 历年真题演练 / 2026 新增题库 / 模拟题 / 模拟训练 / 章节测试 / 习题精选 / 综合模拟题”等题库型区块，将可解析题目写入本地题库，供导出章节题库和学生端章节练习使用；导入必须跳过“课堂提问 / 课堂互动 / 导入提问”等教学互动内容。导入必须兼容 Notion Agent 常见的“多个题目集中在 callout、参考答案集中在后续 details 中”的格式，例如 `1-B 2-C（解析）4-C 5-C`，并按题号把答案和解析回填到对应题目；如果题库区没有单独写“单选题 / 多选题”小标题，也应能根据编号题和 A/B/C/D 选项解析入库；如果题目标题写成 `单选题 1【模拟题】`、`多选题 1【模拟题】`、`判断题 1【模拟题】`、`操作题 1【模拟题】`，或真题小节写成“历年真题单项选择题 / 多项选择题 / 判断题 / 操作应用题”，都必须识别为对应题型并导入。多选答案必须支持连续字母格式，例如 `ABC / ABCE / ABCDE`，同时兼容 `A/B/C`、`A，B，C`、`A、B、C`；判断题必须支持 `√ / × / 对 / 错`；操作题允许没有选项，答案按“按步骤评分”保存，参考步骤保存到解析。导入结果必须按单选、多选、判断、简答、操作显示解析 / 新增 / 更新数量，便于老师发现漏导。导入时必须按核心题干合并旧解析残留题，例如旧题干把 `A. xxx` 混入题干、新题干只保留题目主体时，应识别为同一道题并更新旧记录，不新增重复题。导出章节题库时如果本地题库为空，也应先从最新教学页兜底解析一次。
  - 老师端当前章节必须提供“显示/编辑章节习题”能力，导入当前章节习题后可以查看本地章节题库，并支持新增、编辑、隐藏和恢复题目。可编辑字段包括题型、题干、选项、答案、解析、难度、来源、年份和知识标签。老师点击删除类操作时采用“隐藏归档”而不是物理删除：隐藏题不进入学生练习、模拟考试和导出题库，但保留学生历史答题记录，避免破坏错题回看。
  - 老师端当前章节必须提供“清理当前章节重复题”能力，只清理 `Notion AI` 导入来源的重复题，包括自编题、模拟题和题库题。系统按核心题干分组，保留答案、解析、选项最完整的一条；无学生答题记录的重复题可删除，有答题记录的旧题改为“重复保留”并从普通练习、模拟考试和导出题库中隐藏，避免破坏历史答题记录。
  - C 生成结果需要保留结构校验 warning，确保老师能看到缺少分页、callout、真题折叠解析、教师备课卡等格式问题。
  - 支持手动扫描 Notion 触发项：`自动填充考点`、`入库`、`生成课件`。
  - 教学页先写回 Notion 章节库，再通过“同步当前章节教学页”同步到网页预览 / 展示区。
  - 支持导出 Markdown、教学网页、PPT、章节题库；PPT 导出按专升本《计算机应用基础》课程课件模板生成，包含封面、目录、学习目标、知识结构、重点难点、小节讲解、真题演练、课堂小结、课后作业和教师备课卡；教学网页导出为单文件 HTML，按章节教学网站样式展示 hero、可折叠目录、摘要卡片、模块化 section、表格、Markdown 管道表格、代码 / mermaid、callout 和折叠答案解析；章节题库导出为单文件 HTML，包含题型统计、搜索、题型标签筛选、题卡、选项、难度 / 年份 / 来源 / 知识点标签和折叠答案解析，并按“一、单选 / 二、多选 / 三、判断 / 四、简答 / 五、操作”展示题型分区。

### 教师工作台 `/teacher`
- 学生授权管理。
- 学生账号权限收回与删除。
- 章节内容管理。
- 题库管理。
- 布置练习。
- 学习反馈查看。
- 教学页生成入口。
- 导出课件 / PPT / 教学网页 / 章节题库。

## 6. 第一版范围

### 第一版会做
- 学生登录注册和授权访问。
- 学生按章节学习、查看知识点、下载资料。
- 章节练习、错题回看、模拟考试；答题记录保存在本地 SQLite，便于后续老师查看学习反馈。
- 教师查看教案、下载课件、管理题库、布置练习、查看学习反馈。
- Qwen + Codex Agent + Notion 的教学页生成流程。
- 教学页写回 Notion 章节库，并同步到网页。
- 导出 Markdown、教学网页、PPT、章节题库。

### 第一版不做
- 在线支付。
- 复杂后台系统。
- 学生社交讨论区。
- 学校教务系统。
- 多学校、多班级、多层级组织架构。
- 移动 App。
- 复杂权限体系。

## 7. 目标衡量

第一版上线后 1-3 个月，重点看：

- 学生能否完成登录授权，并按章节完成学习和刷题。
- 老师能否用系统完成备课。
- 至少完成若干章节的教学页生成。
- 教学页能否成功写回 Notion 章节库并同步到网页。
- 能否导出课件 / PPT 并用于课堂。

## 8. 风险与待解决问题

- Notion 数据库结构需要稳定，包括章节库、原始页面库、题库 / 真题库等。
- Qwen 图片转 Markdown 的质量会影响后续教学页质量。
- Codex Agent A/B/C 的输出需要可检查、可回滚，避免错误内容直接进入教学页。
- 目前 Codex Agent A、B 已完成，但 C 还不能实现原 Notion Agent 中的所有功能；第一版需要明确 C 的最小可用范围，以及哪些高级能力暂时保留在后续迭代。
- 学生登录注册和授权访问需要明确账号、密码、审核、重置密码等基础规则。
- 第一版加入登录后，开发复杂度明显高于纯静态课程网站。
- PPT / 教学网页 / 章节题库导出的格式需要后续单独确认。
