# v2 Public Home · 参赛主页与公开试用页

## 背景

教考智联将参加第三十届辽宁省教育教学信息化交流活动的“人工智能+教育案例”方向。公开主页需要同时承担参赛展示、真实体验入口和角色分流入口，让评审、老师、学生和访客快速理解平台价值并能直接试用。

## 目标

- 公开展示教考智联作为专升本《计算机应用基础》AI 教与学工作台的定位。
- 说明“Qwen 原始资料结构化 + A/B/C Notion Agent 生成 + 教师人工审核 + 学生学习闭环”的核心流程。
- 提供无需登录的试用教学页入口，避免访客因没有账号无法体验。
- 提供学生端登录/申请访问和老师端登录入口。
- 用克制、可核验的方式展示应用成效与前期实践基础。

## 页面范围

### `/`

公开主页，所有用户可访问。即使已经登录，访问 `/` 也必须继续显示公开主页，不自动切到老师端或学生端。主页包含：

- 顶部导航：案例简介、AI 工作流、教学闭环、应用成效、体验入口。
- 第一屏：教考智联定位、参赛标识、核心说明和三个入口按钮。
- AI+教育案例亮点：智能助教、智能助学、可控发布。
- A/B/C 智能体工作流：资料上传、A 考点、B 真题、C 教学页、人工审核、发布到学生端。
- 学生学习闭环：进入章节、阅读教学页、完成练习、参加模拟考试、回看错题。
- 老师备课闭环：章节同步、资料上传、A/B/C 生成、审核、授权、导出。
- 应用成效与实践基础：6 章、369 道真题、约 4 小时压缩到约 20 分钟、桌面版参赛一等奖但证书待下发。
- 底部可信说明：课程边界、授权、公开试用、人工审核和竞赛成果表述边界。

### `/trial`

无需登录的公开只读试用章节。固定展示当前数据库 `chapters.id = 26` 的 `第 1 章第 1 节-计算机概述`，读取该章节最新教学页 markdown 和未隐藏、非重复保留的练习题。试用页只用于公开体验，不写入练习记录、错题记录或授权关系。

试用页底部“正式课程会提供什么”使用图文卡片展示章节教学页、历年真题练习、模拟考试、错题回看、资料下载、老师授权开放。前五项使用学生端界面截图，老师授权开放使用项目风格示意图；卡片只做功能说明，不作为无需登录入口。

### `/login?role=student`

学生端登录 / 申请访问入口，复用现有认证接口。

### `/login?role=teacher`

老师端登录入口，复用现有认证接口。首次无老师账号时仍显示初始化老师账号。

### `/teacher`

老师工作台入口。老师登录成功后跳转到该路径。

### `/student`

学生学习首页。已授权学生登录成功后跳转到该路径。

### 工作台返回主页

老师端和学生端侧边栏都提供“返回公开主页”按钮。该操作只导航到 `/`，不退出登录。

## 文案边界

- 主页可使用“曾指导学生基于该作品桌面版参加计算机设计大赛，已获多省联赛一等奖，证书待下发”。
- 证书下发前，不写成已经拿到证书或已经完成证书交付。
- AI 相关内容必须强调教师人工审核后发布。

## 验收标准

- 未登录访问 `/` 能看到参赛主页和三个入口。
- 点击无需登录试用进入 `/trial`，不触发登录，并能看到 `第 1 章第 1 节-计算机概述` 的完整教学页。
- `/trial` 的练习题可在本地选择答案并查看解析，但不新增 `question_attempts` 记录。
- `/trial` 底部功能说明区显示 6 张图文卡片，桌面端 3 列、移动端单列展示，图片、标题、说明不重叠。
- 点击学生入口进入 `/login?role=student`。
- 点击老师入口进入 `/login?role=teacher`。
- 老师登录成功后进入 `/teacher`。
- 学生登录成功后进入 `/student`。
- 已登录用户访问 `/` 仍显示公开主页。
- 工作台点击“返回公开主页”后进入 `/`，且不退出登录。
- 移动端单列展示，按钮和指标不重叠。
- `npm run check` 和 `npm -w apps/web run build` 通过。
